## Project Overview

GeroExplorer is an interactive graph visualization of aging-disease relationships.
Core data model: **Hallmarks of Aging → Mechanisms → Diseases** (all with confidence-scored edges).

See README.md for full project description.

## Architecture

### Data layer (`data/`)
Three JSON source files → assembled into `graph.json` by `scripts/build_graph.py`.

- `hallmarks.json` — list of 12 hallmark objects (all Lopez-Otín 2013 + 2023)
- `mechanisms.json` — list of mechanism objects with `hallmark_links[]`
- `diseases.json` — list of disease objects with `mechanism_links[]` and `hallmark_direct_links[]`
- `graph.json` — D3-ready `{ nodes[], links[] }` — **never edit directly, always rebuild**
- `pubmed_cache.json` — generated by `fetch_pubmed.py`, gitignored

### Data pipeline (`scripts/`)
- `build_graph.py` — run after any data change; also copies output to `frontend/public/graph.json`
- `fetch_pubmed.py` — optional PubMed confidence calibration; respects NCBI rate limits (0.35s/req)

### Frontend (`frontend/`)
React 18 + D3 v7 + Vite 5. No CSS framework — plain CSS with dark theme (#0f172a background).

Key component: `ForceGraph.jsx` — D3 force simulation in SVG, integrated via `useRef`/`useEffect`.
Graph data is served from `frontend/public/graph.json` at `/graph.json`.

## Confidence Scores

Edges carry a `confidence` value (0–1). Current source: manually set based on literature knowledge.
After running `fetch_pubmed.py`, scores are updated with log-scaled PubMed publication counts.

**Open decision:** whether to expand to effect size / study quality weighting. This affects parser complexity significantly. Do not implement until Fredrik decides.

## Important conventions

- IDs use snake_case prefixed by type: `hallmark_*`, `mech_*`, `disease_*`
- Node types in graph.json: `"hallmark"`, `"mechanism"`, `"disease"`
- Edge types: `"hallmark_to_mechanism"`, `"mechanism_to_disease"`, `"hallmark_to_disease"`
- Always rebuild graph.json after editing any data file: `python3 scripts/build_graph.py`
- After rebuilding, `frontend/public/graph.json` is updated automatically

## Development

```bash
# Rebuild data
python3 scripts/build_graph.py

# Frontend dev server
cd frontend && npm run dev

# PubMed calibration (optional, slow ~10-20 min for all nodes)
python3 scripts/fetch_pubmed.py --email your@email.com --dry-run  # preview
python3 scripts/fetch_pubmed.py --email your@email.com             # run
```

## What NOT to do

- Do not edit `data/graph.json` directly — it is generated
- Do not hardcode disease/mechanism data into the frontend — always read from graph.json
- Do not add frontend frameworks (no Tailwind, no MUI) without asking
- Do not over-engineer confidence scoring until the model decision is made
